## Раздел 1 — Структуры, методы, интерфейсы
~~#### Задача 1 — Менеджер задач~~
##### Создать структуры для управления списком задач.
* Добавить методы:
* добавление новой задачи;
* пометка задачи как выполненной;
получение списка всех задач.
Добавить проверку: если задача с указанным ID не найдена — вернуть ошибку.
Реализовать, как будут генерироваться уникальные ID.

~~#### Задача 2 — Магазин товаров ~~
Создать структуры для магазина и товара.
Реализовать методы:
добавление товара в магазин;
поиск товара по названию;
вычисление общей стоимости всех товаров.
Реализовать поведение, если товар не найден.

~~#### Задача 3 — Интерфейс фигур~~
Создать интерфейс, который описывает фигуру с методами для вычисления площади и периметра.
Реализуй несколько типов фигур (например, прямоугольник, круг).
Сделать функцию, которая принимает любую фигуру и выводит её тип, площадь и периметр.

#### Задача 4 — Банковские аккаунты
Создать интерфейс банковского счёта с методами для пополнения, снятия и проверки баланса.
Реализовать два типа счетов:
дебетовый — не может уходить в минус;
кредитный — может уходить в Разминус до определённого лимита.
Продумать, как будут обрабатываться ошибки (например, попытка снять больше, чем есть).
#### Задача 5 — Работа с пользовательским типом
Создать собственный тип на основе среза чисел.
Добавить методы для:
вычисления суммы;
вычисления среднего значения;
фильтрации элементов по условию (принимается функция-предикат).
Проверить работу на разных наборах чисел.
#### Задача 6 — Сервис пользователей
Создать сервис для управления пользователями.
Добавить методы для:
создания нового пользователя;
поиска по ID (с возвратом ошибки, если не найден);
удаления пользователя;
получения списка всех пользователей.
Добавить собственный тип ошибки “пользователь не найден”.
## Раздел 2 — Ошибки
#### Задача 1 — Деление с проверкой
Реализовать функцию деления двух чисел, которая возвращает ошибку, если делитель равен нулю.
Обработать ошибку в вызывающем коде.
#### Задача 2 — Кастомная ошибка
Создать свой тип ошибки, например “не найден элемент с ID”.
Использовать его в функции, которая ищет элемент по ID в коллекции.
Проверь ошибку через errors.As или errors.Is.
#### Задача 3 — Оборачивание ошибок
Создать базовую ошибку, например “ошибка базы данных”.
В другой функции обернуть её через %w.
Проверить в вызывающем коде, что ошибка корректно распознаётся через errors.Is.
## Раздел 3 — REST API (на базе Gin)
Изучить и понять:
Как работает библиотека Gin:
инициализация сервера;
маршруты и группы маршрутов;
запуск сервера.
Как обрабатывать HTTP-методы: GET, POST, PUT, DELETE.
Как читать тело запроса (BindJSON, ShouldBindJSON).
Как формировать ответ в JSON (c.JSON, c.IndentedJSON).
Как возвращать статусы и ошибки (http.StatusOK, http.StatusBadRequest, http.StatusNotFound, http.StatusInternalServerError).
Как получать параметры маршрута (Param) и query-параметры (Query).
Как использовать middleware (логирование, обработка ошибок, CORS).
Как валидировать данные, возвращать осмысленные ошибки и корректные статусы.
